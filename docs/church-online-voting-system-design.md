# 教會長老／執事線上投票系統設計（無記名、QR Code 金鑰登入）

## 1. 目標與範圍

本設計提供一套「不需自行架站也可落地」的線上投票系統藍圖，支援：

- 無記名投票（票與人分離，不可追溯）
- 會友 QR Code 金鑰登入
- 前台（會友）與後台（管理員）分離
- 候選人名單投票與提交前確認
- 後台選舉設定（投票時間、候選人、應選名額、結束投票）

---

## 2. 角色與權限

### 2.1 角色

1. **會友（Voter）**
   - 使用收到的 QR Code 掃描登入
   - 只能進入自己可投的選舉
   - 只能看到候選人與名額，不可看到其他人投票狀態

2. **選務管理員（Admin）**
   - 維護選舉資料、候選人、名額、時間
   - 發放/重發 QR Code 金鑰
   - 可手動提前結束投票
   - 可查看統計報表（不含個人票向）

---

## 3. 無記名投票核心設計

## 3.1 雙表分離（關鍵）

為確保無記名，至少分兩類資料：

1. **登入憑證表（VotingToken）**
   - 僅保存「金鑰是否有效、是否已使用、對應哪場選舉」
   - 不保存投了誰

2. **選票表（Ballot）**
   - 僅保存「選了哪些候選人」與時間
   - 不保存會友身份資訊

> 重點：系統在驗證金鑰有效後，產生一組「不可回推身份的 ballotId」寫入選票，再將原 token 標示為已使用。

## 3.2 QR Code 金鑰規則

- 每位會友每場選舉一個一次性金鑰（單次使用）
- 金鑰建議長度：至少 128-bit 隨機值
- QR Code 內容：
  - 直接連結（例如 `https://vote.example.org/vote?t=<token>`）
  - 或短碼 + 後端兌換機制
- 金鑰過期條件：
  - 超過投票結束時間自動失效
  - 已使用即失效
  - 管理員可手動作廢重發

---

## 4. 前台（會友）流程設計

## 4.1 登入流程

1. 會友掃描 QR Code
2. 系統驗證 token：
   - 是否存在
   - 是否未使用
   - 是否在投票期間
   - 是否屬於正確選舉
3. 驗證通過後進入投票頁

## 4.2 投票頁規則

- 顯示：
  - 選舉名稱（如「2026 年長老選舉」）
  - 候選人清單（可附照片/簡介）
  - 應選名額（例如 5 名）
- 勾選邏輯：
  - 可「少選」（例如名額 5，選 1~5 都允許）
  - 不能「超選」（超過名額立即阻擋）
- 送出前二次確認：
  - 列出已選名單
  - 顯示「送出後不可修改」
  - 按下確認才正式送出

## 4.3 送出後

- 顯示「投票成功」與時間
- token 標示已使用，禁止再次投票
- 不顯示可識別個人之投票紀錄

---

## 5. 後台（管理端）功能設計

## 5.1 選舉基本資料建置

管理員可建立：

- 選舉名稱（長老/執事）
- 選舉說明
- 投票開始時間、結束時間
- 應選名額
- 狀態（草稿 / 進行中 / 已結束）

## 5.2 候選人管理

- 新增/編輯/停用候選人
- 欄位建議：姓名、職稱、簡介、照片（選配）、排序

## 5.3 會友與金鑰管理

- 匯入會友名單（CSV）
- 一鍵產生個人 token
- 下載/列印 QR Code（PDF）
- 狀態追蹤：未使用 / 已使用 / 已作廢
- 支援遺失重發（舊 token 自動作廢）

## 5.4 投票控制

- **開始投票**：到開始時間自動啟用（或管理員手動啟用）
- **結束投票**：
  - 到時間自動結束
  - 管理員可手動提前結束

## 5.5 開票與報表

- 即時票數統計（建議在投票結束後再開放）
- 匯出結果（CSV / PDF）
- 稽核紀錄（誰在何時做了哪個管理操作）

---

## 6. 資料模型（建議）

## 6.1 Election（選舉）

- id
- title
- positionType（elder/deacon）
- seats（應選名額）
- startAt / endAt
- status

## 6.2 Candidate（候選人）

- id
- electionId
- name
- bio
- avatarUrl
- sortOrder
- active

## 6.3 Voter（會友）

- id
- memberNo
- name
- phone/email（選配）
- active

## 6.4 VotingToken（登入金鑰）

- id
- electionId
- voterId
- tokenHash（只存雜湊，不存明碼）
- issuedAt
- expiresAt
- usedAt
- revokedAt

## 6.5 Ballot（選票）

- id（隨機 UUID）
- electionId
- submittedAt

## 6.6 BallotChoice（選票明細）

- id
- ballotId
- candidateId

## 6.7 AuditLog（稽核）

- id
- adminId
- action
- payload
- createdAt

---

## 7. 安全與合規建議

1. **全站 HTTPS**
2. **token 只存雜湊值**（避免資料庫外洩直接被投票）
3. **後台強制 MFA**
4. **速率限制與防機器人**（防暴力嘗試 token）
5. **資料備份**（投票期間至少每日 + 重要操作後備份）
6. **稽核不可竄改**（append-only log）
7. **隱私公告**：明確告知蒐集資料用途與保存期限

---

## 8. 不會架站時的落地方式（建議）

## 8.1 最快可行方案

- 使用雲端平台部署（例如 Vercel / Render / Railway）
- 資料庫使用託管型 PostgreSQL（Neon / Supabase / RDS）
- 網域與 HTTPS 由平台處理

## 8.2 交付物

1. 前台投票頁（手機優先）
2. 後台管理頁（電腦優先）
3. QR Code 批次產生器
4. 結果報表匯出
5. 管理手冊（一步一步操作）

---

## 9. 介面草圖（文字版）

## 9.1 前台投票頁

- Header：選舉名稱 + 截止時間
- Body：候選人卡片（姓名、簡介、勾選框）
- Footer：
  - `已選 3 / 應選 5`
  - `確認送出`按鈕（超選則 disabled）

## 9.2 送出確認彈窗

- 標題：請再次確認您的選票
- 內容：已勾選候選人名單
- 提示：送出後無法修改
- 按鈕：返回修改 / 確認送出

## 9.3 後台管理頁

- 側欄：
  - 儀表板
  - 選舉設定
  - 候選人管理
  - 會友與金鑰
  - 投票控制
  - 統計報表
- 主區塊：資料表格 + 新增/編輯抽屜視窗

---

## 10. 驗收清單（UAT）

1. 掃描 QR 可正常進入投票頁
2. 無效/過期/已用 token 會被阻擋
3. 可少選，不可超選
4. 送出前有二次確認
5. 送出後不可重投
6. 後台可設定開始/結束時間
7. 後台可手動結束投票
8. 報表只顯示統計，不可回推個人投票
